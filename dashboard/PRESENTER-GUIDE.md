## 三、三大面板 (卡片) 详细说明

### 3.1 AI INTENT STREAM (左侧 30%)

| 元素 | 说明 |
|------|------|
| **标题** | "AI INTENT STREAM" |
| **绿色圆点** | 实时连接状态指示器 |
| **搜索框** | 按 action/amount/ID 过滤终端日志 |
| **列表列头** | Time · ID · Action · Amount · Status |
| **日志行** | 每行 = 一个 AI 生成的意图。行按 action 类型显示不同颜色 |
| **底部汇总** | AVG SIZE(平均交易量)、BATCHED(已批处理数)、TOTAL(总量) |

**演示话术**：
- "左侧是 AI 意图终端。每行代表 DeepSeek V4 生成的一条意图。"
- "灰色的 pending 表示正在队列中等待打包，绿色的 BATCHED 表示已上链。"
- "点击任意行→弹出 AI 推理详情：可以看到 DeepSeek 为什么做这个决策。"

### 3.2 BATCHING ENGINE (中间 40%)

| 元素 | 说明 |
|------|------|
| **标题** | "BATCHING ENGINE" |
| **顶部说明栏** | "10 次独立交易需 ~120K gas/次，打包 10 个为 1 笔可省 ~70% gas" |
| **圆环** | 被填充的液态桶。粒子围绕轨道旋转 |
| **中心数字** | 当前队列中意图数量 / 批处理阈值(10) |
| **进度条** | 填充到 100% 时触发批处理 |
| **流程指示器** | AI Intents → Queue → Batch → On-Chain |
| **底部信息** | 上次批处理的结果(batchId, 意图数, 节省百分比, tx hash) |

**演示话术**：
- "中间是批处理引擎的核心——意图队列。每注入一条，液态桶的液位就上升一格。"
- "当桶满 10 个→自动触发链上执行。右下角会弹出 BatchExecuted 弹窗。"
- "批处理的魔法：10 个意图打包进 1 笔交易，base cost 从 10 份降为 1 份。"

### 3.3 PERFORMANCE METRICS (右侧 30%)

| 元素 | 说明 |
|------|------|
| **标题** | "PERFORMANCE METRICS" |
| **三个模式按钮** | GAS COMPARE / THROUGHPUT / CUMULATIVE |
| **ECharts 图** | 实时渲染的折线图/柱状图 |
| **三个统计数字** | BATCHES(总批次数)、INTENTS(总意图数)、GAS SAVED(总节省 Gas) |
| **Recent Batches 表格** | 最近 15 个批次的逐条 Gas 节省百分比 |

**ECharts 模式说明**：

| 模式 | 图表类型 | 展示内容 | 演示话术 |
|------|---------|---------|---------|
| **GAS COMPARE** | 双线图 | 红线=传统模式(独立交易)，绿线=批量模式(1 笔 tx) | "绿线远低于红线——这就是批处理的力量。" |
| **THROUGHPUT** | 柱状图 | 每批次的意图数量 | "看每个柱子的高度：每批都稳定处理 10 个意图。" |
| **CUMULATIVE** | 面积图 | 累计 Gas 节省曲线 | "这条向上的曲线——是累积的 Gas 节省。时间越长，省得越多。" |

## 四、弹窗与模态

| 弹窗 | 触发方式 | 说明 |
|------|---------|------|
| **Batch Executed Popup** | 链上批次确认时自动弹出 | 显示 batchId、意图数、Gas 对比、节省百分比。内含跳动粒子动画。5 秒自动消失。 |

## 六、常见问题

### Q: 需要提前启动哪些服务？
```
终端 1: npx hardhat node           (区块链)
终端 2: npm run deploy:local       (部署合约)
终端 3: npm run approve:local      (授权)
终端 4: cd relayer && npx ts-node src/index.ts  (Relayer)
终端 5: cd dashboard && npx vite --host          (前端)
```

### Q: INJECT 没反应怎么办？
A: 检查 Terminal 输出的日志。如果显示 "accepted" 则正常。检查 Relayer 终端是否有 `[Queue]` 日志。

### Q: 弹窗不出来？
A: 需要等队列满 10 个后链上确认才触发。可以检查 GET /batches API 是否有新批次。

### Q: 市场行情显示 Network Error？
A: CoinGecko 免费 API 有限流。等待 60 秒自动恢复。不影响其他功能。

### Q: MetaMask 连接报错？
A: 演示不需要 MetaMask。跳过即可。如果要用需要到 `https://localhost:5173` (HTTPS)。

### Q: 没有 DeepSeek API Key，AI 还能工作吗？
A: 能。Random 模式离线可用，会随机生成 swap/dca/rebalance/compound/arbitrage 五种意图，效果一样好。
